Frequently Asked Questions

1. What is the difference between var, let, and const?
In JavaScript, var, let, and const are used for variable declaration but have key differences. var is function-scoped, allows re-declaration, and is hoisted with undefined, making it less reliable. let is block-scoped, does not allow re-declaration, but can be reassigned. const is also block-scoped but cannot be reassigned, making it ideal for constants. While var is outdated and can cause issues, let is useful for mutable values, and const is the preferred choice for most cases. Best practice is to use const by default and let only when necessary.
2. What is the difference between map(), forEach(), and filter()
In JavaScript, map(), forEach(), and filter() are array methods used for iterating over elements but serve different purposes. map() creates a new array by applying a function to each element, making it useful when transformation is needed. forEach() simply executes a function for each element without returning a new array, making it ideal for performing side effects like logging or updating values. filter() creates a new array containing only elements that pass a given condition, making it useful for data filtering. While map() and filter() return new arrays, forEach() does not, so map() is preferred when transformation is needed, filter() for extracting specific elements, and forEach() for operations without needing a returned array.
3. What are arrow functions and how they are different from regular functions?
Arrow functions in JavaScript are a shorter and more concise way to write functions using the => syntax. Unlike regular functions, arrow functions do not have their own this value; instead, they inherit this from their surrounding scope, making them useful in callbacks and event handlers. They also do not have their own arguments object and cannot be used as constructors with new. Regular functions, on the other hand, have their own this, making them more flexible for object methods and dynamic context binding. Arrow functions are best for short, simple functions, while regular functions are better when this or arguments is needed.
4. How JavaScript Promises work?
JavaScript Promises handle asynchronous operations by representing a future value with three states: pending, fulfilled, and rejected. They use .then() to handle success, .catch() for errors, and .finally() for cleanup. A Promise is created using new Promise((resolve, reject) => { ... }), where resolve() is called on success and reject() on failure. Promises prevent callback hell and allow chaining for cleaner, more readable code.
5. How closures work in JavaScript?
A closure in JavaScript is a function that retains access to variables from its outer (enclosing) function, even after the outer function has finished executing. This happens because the inner function "remembers" the scope in which it was created. Closures are useful for data encapsulation, maintaining state, and handling callbacks.
